# Data Concept 

We have introduced the ability to assign the corresponding Data Concept that belongs to your API while publishing or updating. Simply add "dataConceptName": "value" to the request body.

## Definition

Data concepts are the information artefacts that describe the data used in processes.  They are used for finding and understanding data at a conceptual level within process context and means for communication. It sets the boundaries/ scope of data to be detailed out at logical and physical level after which it can be implemented into systems.

Data concepts distinguish different levels of abstraction and require a taxonomy to capture relations amongst them. They are not pointing to logical data entities or attributes. Data concepts are unique and facilitate re-use of data across the IT landscape. All data concepts must be defined with terms and described in the business glossary. Terms within the business glossary can reflect many artefacts at different levels, thus are not always data concepts.

## Why do we need Data Concept and what is the value

- Data Concepts help establish common vocabulary to enable effective communicate in system agnostic language across the different parts (technical/Non-Technical) parts of the Organisation
- Data Concepts can also serve as Input and Output to the Business process modelling at various levels of processes.
- Data Concepts helps in the software design process and System Implementation.
- Data Concepts is key building block for Data Democratisation through Data Marketplace.
- A collection of Data Concept can also be used to monitor and govern the KPIs on the Data quality, Data classification or Data governance of company data asset.
- Data Concepts contribute to Interoperability, i.e. the "I" in F.A.I.R. By using the same name and definition for a piece of data across domains, we will understand each other. By designing business API:s  according to naming and meaning of Data Concepts, our systems can communicate in an interoperable way.
- Data Concepts help in the communication of responsibilities for data. One digital domain is responsible for certain Data Concepts, another domain is responsible for other Data Concepts. It must be clear who is responsible for what data.
