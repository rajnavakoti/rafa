- The purpose of creating this document is to provide a template to the provider which they can use to define the Generic information about the API and wants the API Consuemrs to be awared of. This document can be created as md file and Github url of document can be passed as one of the property(apiGenericInfoDoc) during deployment of an API.Once deployed this document will be visible as one of the tab ApiGenericInfo on the API detail page of Allen.

### Source Code of Template page

The Template is wrapped here in souce code so that provider can copy the template code in thier own md file and start fill in their own details.

## More description about API

This section can be used by Provider to give more description about the API like: when api is developed.How many versions of API is available for use. which version is available in which environment etc.

## Basic rule for subscription and API call

This section can be used by Provider to describle the Basic Rule that consumer of the API has to follow for subscription of the API.

## How to make API call

This section can be used by Provider to describe how to make the call of the API like: giving sample code,given curl command , post-man request template etc.

## Best and Recommended practice to use the API

This section can be used by Provider to describe what the best and recommended way to use the API.

## How to communicate new or depricated version of API

This section can be used by Provider to how the Consumers will be informed about new ,depricated or retired version of your API. This section can also be used to define for a Consumers on how to use your draft version of your API to get their feedback before actual publishing.

## How to find Logs of API

This section can be used by Provider to describe the Consumers how and where they can see the logs of the api call.

## How to find metrics of API

This section can be used by Provider to provide the link to the page where Consumer can see the matrics of the pulished api like: average how many calls made per day, how many success, how many failed, average response time etc..

## How to find SLA Data of API

This section can be used by Provider to describe the where and how to check the SLA data like: Total quota, Used quota, Remaining Quota, Response time etc.

## Popular use cases of API

This section can be used by Provider to describe the popular use cases used by different Consuemers for the API published. This section can also be used to give sample payloads,codes etc for those popular use cases.

## How to report an issue related to API

This section can be used by Provider to describe the Consumers how and where to report the issues related to API like: api not responding,api-latency issue, api seruirity concern etc.

## How to know any planned outage

This section can be used by Provider to describe the Consumers where and how they can be informed about any planned outage of an api. This section can also describe if any corrective measures consumer has to take during down time etc.

## Future Road Map

This section can be used by Provider future road map of their api. This can also define the current limitation of any and also how and when it is planned to be handled in future.

## Contact Us

This section can be used by Provider to give any contact information of their team. This can also be used to give link to the feedback page or sample template to give feedback in mail etc.
